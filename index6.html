ABOUT

This project involves the implementation of a cybersecurity test bench aimed at optimizing communication in a vehicular mobility context. The objective is to design and develop a secure platform for communication using the MQTT protocol between autonomous vehicles and fixed stations. The platform integrates TLS (Transport Layer Security) for encryption and mutual authentication using certificates, ensuring a secure and reliable communication channel.

The project focuses on developing a secure communication interface between vehicles and control stations, ensuring that data is transmitted safely and efficiently. It also involves analyzing performance and latency, with a specific focus on encryption algorithms and key sizes to optimize security without compromising speed.

Additionally, the project includes the design of a graphical interface that allows for easy configuration and real-time data visualization, making the system both user-friendly and efficient.

The platform is built using a variety of cutting-edge tools and technologies. ReactJS is used for developing the frontend of the application, creating an interactive user interface. Node.js serves as the backend framework, handling the communication logic, while Prisma is utilized for efficient database management. For scripting and automation, Python is employed, with Docker providing containerized environments for deployment. The project follows the Scrum methodology for effective project management and uses Git for version control and collaboration.

Secured Communication Architecture for Autonomous Vehicles

This image illustrates the architecture of a cybersecurity test bench designed to optimize communication in the context of autonomous vehicles. At the heart of the system is the MQTT server, which acts as a central hub facilitating real-time data exchange, including geolocation and speed information. Vehicles communicate with the server by subscribing to relevant topics, allowing them to publish and receive critical data. To secure these communications, multiple layers of protection are implemented. TLS (Transport Layer Security) ensures encrypted data transmission, while Mutual Authentication via Certificates and Password-Based Authentication protect against unauthorized access. A Firewall further strengthens security by filtering traffic and blocking potential threats. The image also highlights the Study of Latency, where the impact of encryption key sizes on latency is analyzed. Data flows between the server and vehicles (Cars A and B) are tested with different authentication key sizes to optimize performance without compromising security. This latency analysis helps ensure minimal delays in data transmission, a critical factor for real-time decision-making in smart city environments.

Test Interface for Connection and Latency Analysis

This interface is used to run tests on the communication between the MQTT server and autonomous vehicles. The user can select different encryption keys (e.g., RSA_256, RSA_512, RSA_1024) for both the client and server sides to analyze the impact of key sizes on the connection. After selecting the desired keys, the "Launch Test" button initiates the test, during which the system measures various metrics, such as connection time and latency. On the right side, the results are displayed in a graphical format, showing the connection times for different key configurations. This visual feedback helps in evaluating the performance and efficiency of the encryption protocols being tested. The user can stop the test at any time using the "Stop Test" button. This interface is essential for optimizing the security and performance of communication in the autonomous vehicle network.

Latency Test Interface for Key Size Impact Analysis

This interface allows users to perform latency tests to evaluate the impact of different encryption key sizes on the authentication process. The user can initiate the test by clicking the "Launch Latency Test" button, after which the system measures the latency for various key sizes, such as RSA_256, RSA_512, RSA_1024, Curve448, and P-256. The results are displayed on the right side of the interface, where a bar chart shows the correlation between key size and authentication latency. As shown in the graph, larger key sizes typically result in higher latency, providing valuable insights for optimizing the balance between security and performance in the autonomous vehicle communication network. The test can be stopped at any point by clicking the "Stop Latency Test" button. This test is critical for determining the most efficient encryption configurations, ensuring both secure and real-time communication in a smart city mobility context.
